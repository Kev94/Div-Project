Five Ways a Programmer Should Think:

Procedural Thinking:

A programmer should break problems down into steps/procedures to solve problems more efficiently.

Abstract Thinking:

Essentially finding similarity/generality among seemingly differentthings.

Systematic Debugging:

Creates an opportunity for a programmer to identify problems, find the causes of such problems, and finally fix those problems.

Technological Empathy

The computer is written in a different language, called machine language that is different than human language. So one must understand how the computer sees things to be able to communicate properly with it.

Systems Thinking

Lastly, a programmer should take a big idea and break it down into it's component pieces to code efficiently and correctly.

HTML Basics
Tags and Elements:

A tag allows for the altercation of a html document. For example, if I wanted to bold the rest of the sentence I would use a bold tag or if I wanted to italicize something, I would use the emphasis tag. If I wanted to create a link to another page for additional information I would use the a tag. You must close the tags so that they dont affect the whole document, unless it's a void tag, which does not have any content. For example, image tags and break tags are void tags.

Another important concept in html are container elements, which are elements that have no other purpose but to contain stuff within them. There are two types of elements: Inline and Block. Inline elements and tags include:
span, br, em, img, a, and strong. These inline elements and tags keep everything on the same line, except for br.
On the contrary, block elements and tags are more of space occupying objects. They include:
form, p, and div
Blocks are more for seperation of objects and basically draws an invisible container around the element or tag.
